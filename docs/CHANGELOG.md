# История изменений

## v2.0 — Стабильное автозаполнение таблицы

- Режим релиз‑таблицы: сбор связанных задач (Relates), классификация стенда и типа задачи.
- Генерация краткого описания каждой задачи через LLM по шаблону `prompt_templates/issue_short_summary.txt`.
- Сортировка по стендам и типам (MS → Оба/Оба? → WCE; Фичи сверху, Баги ниже).
- Формирование стабильной Jira‑таблицы: ссылка только на ключ `[KEY|URL] Название`.
- Идемпотентное обновление: замена только блока таблицы, сохранение «Шаблоны» и «Мануал», без изменения текста вне таблицы.
- Санитизация ячеек: переносы → `\\`, экранирование `|`, снятие лишних кавычек.
- Инференс ключа целевой задачи из родителя при числовом `TARGET_ISSUE_KEY` (например, `7382` → `MSP-7382`).
- Расширенные логи INFO/DEBUG.

## v0.1 — Первый релиз

- Получение данных задачи из Jira (summary, description, comments) с авторизацией PAT/basic и выбором версии API.
- Формирование промпта по шаблону Jinja2 `prompt_templates/release_summary.txt`.
- Вызов локального OpenAI‑совместимого LLM; поддержка `/v1/chat/completions`.
- Улучшенная обработка ошибок и логирование: превью запросов/ответов, сырой JSON Jira в DEBUG, тело ответа при ошибках.
- Управление уровнем логирования через `.env` (`LOG_LEVEL`) с переопределением флагом `--log-level`.
- Добавлен `.gitignore` для Python, виртуального окружения и локальных файлов окружения.
