# DocsStudio

`DocsStudio` - интерфейс для полуавтоматической работы с релизной документацией и проверками документации.

Подробная спецификация интерфейса: `docs/INTERFACE_SPEC.md`.

## Что умеет интерфейс

- Заполнять релизную таблицу в Jira по задачам релиза
- Генерировать готовый changelog-текст для вставки в исходный код документации
- Генерировать одну краткую релизную строку по одной задаче
- Показывать preview релизной таблицы без записи в Jira
- Сравнивать эталон и сгенерированные формулировки
- Проверять очистку чувствительных данных
- Сканировать документацию на стоп-слова
- Экспортировать документацию в набор очищенных файлов для RAG

## Быстрый старт

1. Установите Python 3.9+.
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Создайте `.env` на основе `env_example` и заполните его.
4. Запустите UI:

```powershell
.\.venv\Scripts\python.exe -m streamlit run ui/app.py
```

## Что заполнить в `.env`

Минимально для работы:

- `JIRA_BASE_URL`
- `JIRA_AUTH_TYPE`
- `JIRA_API_TOKEN`
- `LLM_ENDPOINT`
- `LLM_MODEL`

Если используется `basic`-авторизация в Jira, дополнительно нужен `JIRA_EMAIL`.

## Основные сценарии в UI

### 1. Заполнение релизной таблицы

Вход:
- `release parent` - релизная задача с задачами релиза
- `release target` - задача, где находится таблица для заполнения

Результат:
- таблица в `release target` обновляется краткими релизными формулировками
- ручные колонки `Шаблоны` и `Мануал` сохраняются

### 2. Генерация changelog-текста

Вход:
- `release target` с уже заполненной релизной таблицей

Результат:
- готовый текст для вставки в `releases.md`
- условные блоки для `WCE` и `MS`
- специальные WCE-блоки по формальным эмодзи-маркерам

### 3. Проверка документации на стоп-слова

Вход:
- стартовый URL документации
- путь к словарю стоп-слов
- задержка между запросами
- лимит страниц

Результат:
- сводка по проверке
- список проблемных страниц
- контексты найденных вхождений

### 4. Экспорт документации для RAG

Вход:
- стартовый URL документации
- папка для экспорта
- задержка между запросами
- лимит страниц

Результат:
- статьи сохраняются отдельными файлами с сохранением структуры каталогов
- сохраняются заголовки, списки и ссылки
- визуальное форматирование очищается
- рядом создается `manifest.json`

CLI-пример:

```bash
python stop_words_checker-main/doc_exporter.py \
  --url https://docs.example.com/ru/ \
  --output-dir exports/docs_corpus
```

## Дополнительные режимы

В UI также доступны:
- генерация одной релизной строки
- preview релизной таблицы без записи в Jira
- compare эталон vs сгенерированное
- sanitizer check
- история запусков

Для CLI-режимов см. `docs/MODES.md`.

## Ограничения

- Режимы, которые изменяют описание Jira-задачи, работают при wiki-формате описания Jira Server/DC.
- Для Jira Cloud ADF может потребоваться отдельная реализация обновления описания.
