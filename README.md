# Docs Automation

> **Правила и процессы работы с документацией переехали в репозиторий `doc_diplo`, папку `.agent/`** (точка входа — `doc_diplo/AGENTS.md`). Здесь — **только инструмент** автоматизации (релизная таблица, парсинг Jira, генерация changelog, сканер стоп-слов).

Репозиторий содержит инструменты для автоматизации работы с документацией продукта.

## Основные процессы документации

Документационная работа делится на два основных процесса.

### 1. Актуализация документации релиза

Процесс связан с выпуском релиза продукта и включает:
- подготовку релизных заметок
- заполнение релизной таблицы в Jira
- фиксацию задач, которые требуют создания или изменения статей
- согласование таблицы релиза
- перенос согласованных релизных заметок в `releases.md`

Релизная таблица хранится в задаче вида `✒️ Актуализация документации релиза`. В ней для каждой задачи релиза указывается краткое описание пользовательского изменения и отмечается, нужна ли актуализация документации в колонке `Мануал`.

### 2. Создание и изменение статей

Процесс начинается, когда по релизной таблице или отдельной Jira-задаче понятно, что нужно обновить пользовательскую документацию.

Он включает:
- определение нужной статьи или создание новой
- подготовку изменений в `doc_diplo`
- публикацию на дев-стенд
- комментарий в Jira со ссылками на измененные статьи
- согласование и публикацию в production

## Инструменты репозитория

### DocsStudio

`DocsStudio` - Streamlit-интерфейс для полуавтоматической работы с документацией.

Через интерфейс можно:
- заполнить релизную таблицу в Jira по задачам релиза
- сформировать готовый текст релизных заметок для вставки в `releases.md`
- сгенерировать краткое описание по одной Jira-задаче
- сравнить ручные и автоматически сгенерированные формулировки
- проверить документацию на стоп-слова
- экспортировать документацию в очищенный корпус файлов для RAG

Запуск:

```powershell
# Из корня клонированного репозитория
.\.venv\Scripts\python.exe -m streamlit run ui/app.py
```

### DocAgent

`DocAgent` - набор CLI-инструментов, промптов и инструкций для агентных систем, таких как Codex или Claude Code.

Агентный сценарий нужен, когда требуется не просто нажать кнопку в интерфейсе, а довести задачу до результата:
- заполнить релизную таблицу
- проверить структуру и сортировку таблицы
- сверить спорные формулировки с Jira-задачами
- проставить колонку `Мануал`
- определить и обновить статьи по Jira-задаче
- подготовить ветки и merge request для документации
- перенести согласованную таблицу в `releases.md`

Основная инструкция для агентного сценария:

```text
docs/AGENT_WORKFLOWS.md
```

## Структура

- `main.py` - CLI-точка входа для режимов `doc_agent`
- `release_notes.py` - логика релизных таблиц, сортировки, генерации changelog
- `prompt_templates/` - промпты для генерации и проверки формулировок
- `ui/app.py` - интерфейс `DocsStudio`
- `docs/` - технические описания режимов и возможностей
- `stop_words_checker-main/` - проверка документации на стоп-слова и экспорт корпуса
- `docs/AGENT_WORKFLOWS.md` - рабочие сценарии для агента

## Ключевой принцип

`DocsStudio` помогает быстро выполнить отдельные операции вручную через интерфейс.

`DocAgent` описывает агентный процесс, в котором модель понимает контекст документации, использует инструменты репозитория и доводит задачу до состояния, удобного для проверки техписом.
