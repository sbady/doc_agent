# Процесс актуализации документации релиза

## Общее правило

Работа с документацией релиза привязана к фактическому релизу продукта.

## Этапы

1. На выходе релиза создается задача по шаблону `✒️ Актуализация документации релиза`.
2. В задаче используется релизная таблица (шаблонная структура для задач релиза).
3. Для каждой релизной задачи в таблицу добавляется краткое описание изменений.
4. В колонке `Мануал` помечаются задачи, требующие изменения существующих или создания новых статей.
5. Таблица проходит согласование с ответственными (обычно PM/аналитик).
6. После согласования таблица используется как источник для релизных заметок в разделе `Релизы`.
7. По отмеченным пунктам создаются задачи `✒️ Создание/изменение статьи`.

## Результат этапа

- Согласованная релизная таблица.
- Подготовленные релизные заметки.
- Сформированный пул задач на обновление документации.
