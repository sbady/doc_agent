# Dion.Диск dion-1.27.1 - релизные заметки (пользовательские)

В релизе dion-1.27.1 исправлены ошибки в работе поиска, запросов миграции, публичных ссылок и интерфейсов доступа к Дион.Диску. Также обновлены отдельные элементы интерфейса авторизации и улучшены сообщения об ошибках, чтобы пользователю было понятнее, что происходит в системе.

## Что нового

1. Обновлен логотип Дион.Диска на странице новой авторизации

   На странице новой авторизации скорректировано отображение логотипа Дион.Диска. Изменение делает экран входа визуально аккуратнее и единообразнее с интерфейсом продукта.

## Исправления

1. Исправлена работа запросов миграции и результатов поиска

   В таблице запросов миграции теперь корректно отображается значение в поле "От": пользователь видит отправителя запроса, а не получателя. Также исправлено отображение длинных списков: при прокрутке нижняя часть списка больше не выходит за пределы экрана, а на странице результатов поиска доступен скролл.

2. Исправлен текст события при приглашении в общую папку

   В разделе событий скорректирован текст уведомления о приглашении в общую папку. Теперь сообщение понятно описывает действие и название папки, к которой пользователь получил доступ.

3. Исправлена работа публичных ссылок

   Для уже созданных публичных ссылок с паролем убраны кнопки копирования и отображения пароля, которые не могли работать после создания ссылки. Для публичных ссылок с правом редактирования исправлены действия с файлами без предпросмотра: кнопки "Скачать" и "Удалить" теперь выполняют выбранное действие.

4. Исправлены сценарии открытия Дион.Диска и публичных ссылок без доступа к Диску

   Если пользователь авторизован в Дионе, но ему не подключен Диск, при переходе по прямой ссылке на Диск он перенаправляется на главную страницу Диона. При переходе по публичной ссылке такому пользователю отображается интерфейс публичной ссылки. Для неавторизованных пользователей прямые ссылки на Дион.Диск также обрабатываются корректно и не открывают отдельную форму авторизации Диска.

5. Исправлено отображение квоты пользователя

   В интерфейсе Дион.Диска теперь корректно отображается квота конкретного пользователя. Пользователь видит данные по своему доступному объему, а не некорректное значение, связанное с компанией.

6. Добавлено понятное сообщение при создании папки с уже существующим названием

   Если пользователь пытается создать папку с названием, которое уже используется, система показывает ошибку "Папка с таким названием уже существует". Это же поведение применяется при создании папки в сценарии переноса.

7. Исправлена проверка уникальности идентификатора компании в админке

   При работе с компаниями добавлена проверка уникальности `cust IAM ID`. Если значение уже используется, система сообщает об ошибке и помогает избежать дублирования идентификаторов.
