# Project Docs

Эта папка содержит базовый контекст по проекту документации и рабочие правила.

Точка входа всего репозитория — `AGENTS.md` в корне. Канон процесса по статьям и релизам — `docs/AGENT_WORKFLOWS.md`.

## Структура

- `00-START-HERE.md` — руководство для нового техписа: маршрут чтения и три главных сценария.
- `01-project-context.md` — карта проекта, репозиториев и продуктовых контуров.
- `02-release-documentation-process.md` — процесс актуализации документации релиза.
- `03-article-workflow-and-qa.md` — процесс создания/изменения статей и согласований.
- `04-doc-agent-reference.md` — краткая справка по инструменту `doc_agent`.
- `05-collaboration-with-assistant.md` — как работать с ассистентом в этом проекте.
- `06-subagent-architecture.md` — когда и как привлекать субагентов.
- `07-session-workflow-with-assistant.md` — модель работы в сессии.
- `08-feedback-and-improvement.md` — петля обратной связи и самоулучшения системы.
- `CHANGELOG.md` — журнал изменений правил и методологии.
- `task.md` — типовые рабочие задачи и пошаговые сценарии их выполнения.

## Где что лежит

- Исходники статей: `doc_diplo`
- Инструменты автоматизации: `doc_agent`
