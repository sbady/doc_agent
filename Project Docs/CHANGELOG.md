# CHANGELOG правил проекта

Журнал изменений правил и методологии. Каждая строка: дата — что изменилось — почему — какие файлы. Ведётся по протоколу `08-feedback-and-improvement.md`.

## 2026-06-23

- **Новый формат Jira-комментария.** Заголовок «Проверить можно на dev-стендах:»; каждая статья — отдельной строкой без маркера: `Название — [WCE|dev-url] / [MS|dev-url] — что изменено`; разделитель — длинное тире «—». Переопределяет прежние правила («на дев-стенде», короткое тире «–»). Файл: `docs/AGENT_WORKFLOWS.md`.
- **MR в `main` больше не создаётся по умолчанию.** По умолчанию — только MR в `DEV`; выкатка в `main` (ветка + MR) — отдельный шаг, инициируемый пользователем при согласовании (иначе main-MR висит открытым весь цикл). Файлы: `AGENTS.md`, `docs/AGENT_WORKFLOWS.md`, `skills/git-workflow/*`.
- **Формулировки в статьях — вне времени.** Тело статьи фиксирует факт («доступна печать бейджа»), без «теперь/уже/стало доступно». «теперь» — только для релизных заметок. Файлы: `docs/AGENT_WORKFLOWS.md`, `skills/doc-writer/references/style-guide.md`.
- **Обязательная проверка ссылок между статьями.** Если статья ссылается на другую — открыть связанную и проверить, не устарела ли там информация из-за правки; актуализировать. Файлы: `AGENTS.md`, `docs/AGENT_WORKFLOWS.md`.
- **Сначала изучить, как работает функционал** (цепочка связанных Jira-задач + полнотекстовый поиск + `toc.yaml`), потом решать, что править. Файлы: `AGENTS.md`, `docs/AGENT_WORKFLOWS.md`.
- **Трассировка в коммитах/MR.** Указывать, какие статьи изменены и в рамках каких задач (`MSP-XXXX`). **Без авторства инструментов** (`Co-Authored-By`, Claude, Codex). Файлы: `AGENTS.md`, `docs/AGENT_WORKFLOWS.md`, `skills/git-workflow/*`.
- **Петля обратной связи уточнена.** Фиксируем не любое замечание, а только процессное и повторяющееся; агент предлагает коротко и без воды, и без явного подтверждения владельца ничего в правила не пишет. Файл: `Project Docs/08-feedback-and-improvement.md`.

## 2026-06-22

- Зафиксирован **Definition of Done по умолчанию** для задач-правок: публикация в `DEV` + MR-ссылка без запроса, зеркало в `main` + MR, проверка конфликтов в обе стороны, готовый Jira-комментарий. Почему: повторяющаяся проблема — агент работал в одной ветке, не давал ссылку и комментарий, пользователю приходилось переспрашивать. Файлы: `AGENTS.md`, `docs/AGENT_WORKFLOWS.md` (шаги 13–15).

Скилл явно подчинён канону. Файлы: `skills/git-workflow/SKILL.md`, `references/rules.md`, `references/safe-sequence.md`, `references/task-contract.md`, `agents/openai.yaml`.
