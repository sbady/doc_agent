# CHANGELOG правил проекта

Журнал изменений правил и методологии. Каждая строка: дата — что изменилось — почему — какие файлы. Ведётся по протоколу `08-feedback-and-improvement.md`.

## 2026-06-22

- Добавлена единая точка входа `AGENTS.md` (корень) и `CLAUDE.md` — чтобы Codex и Claude ориентировались одинаково и видели канон. Почему: у свежих агентов не было точки входа, они не знали, что `AGENT_WORKFLOWS.md` каноничен. Файлы: `AGENTS.md`, `CLAUDE.md`.
- Зафиксирован **Definition of Done по умолчанию** для задач-правок: публикация в `DEV` + MR-ссылка без запроса, зеркало в `main` + MR, проверка конфликтов в обе стороны, готовый Jira-комментарий. Почему: повторяющаяся проблема — агент работал в одной ветке, не давал ссылку и комментарий, пользователю приходилось переспрашивать. Файлы: `AGENTS.md`, `docs/AGENT_WORKFLOWS.md` (шаги 13–15).
- Переписан скилл `git-workflow` под модель `main`+`DEV` и публикацию/MR по умолчанию; устранено противоречие со старой консервативной формулировкой («не пушить без запроса»). Скилл явно подчинён канону. Файлы: `skills/git-workflow/SKILL.md`, `references/rules.md`, `references/safe-sequence.md`, `references/task-contract.md`, `agents/openai.yaml`.
- Добавлены `Project Docs/00-START-HERE.md` (онбординг преемника) и `Project Docs/08-feedback-and-improvement.md` (петля самоулучшения).
- **Реструктуризация:** правила и регламенты перенесены в репозиторий `doc_agent` (отдельный от контента `doc_diplo`). `AGENTS.md`, `CLAUDE.md`, `Project Docs/`, `skills/` теперь живут в `doc_agent`; точка входа = корень `doc_agent`. Контент `doc_diplo` (GitLab) подключается рядом как `../doc_diplo`. Пути в документах приведены к корню `doc_agent`. Добавлен раздел про подключение к GitLab.
- Зафиксирован факт **двух продуктов**: MS (есть платные мероприятия) и WCE (платности нет); различия — условными блоками по `domen`. Файлы: `AGENTS.md`, `Project Docs/01-project-context.md`.
- Описан **сканер стоп-слов** (`stop_words_checker-main/`) и правило: прогонять регулярно и обязательно при правках, затрагивающих платную функциональность / термины словаря, чтобы платность не утекала в WCE. Файлы: `Project Docs/04-doc-agent-reference.md`, `AGENTS.md`, `docs/AGENT_WORKFLOWS.md`.
